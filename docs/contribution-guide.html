Contribution guide

A simple contribution model for future TrimTechBR projects and documentation improvements.

How to contribute

  1. Open an issue describing the improvement, bug or new idea.
  2. Create a branch using a clear name, such as feature/add-new-report-section.
  3. Submit a pull request with a concise description and screenshots when relevant.
  4. Keep documentation updated with every functional change.

Standards

  • Prefer clear, practical documentation.
  • Keep scripts and examples reusable.
  • Avoid hardcoded tenant-specific information.
  • Use consistent naming and folder structure.
