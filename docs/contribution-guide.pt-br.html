Setup de desenvolvimento

git clone https://github.com/TrimTechBR/azure-estate-report.git
cd azure-estate-report
Install-Module Az.Accounts, Az.ResourceGraph -Scope CurrentUser
Connect-AzAccount
Import-Module ./AER.psd1 -Force
Invoke-AerReport -OutputPath ./output-test   # ./output-test está no gitignore
Dica: o PowerShell faz cache dos módulos importados. Após editar qualquer .ps1, rode novamente Import-Module ./AER.psd1 -Force (ou use um terminal novo) antes de regenerar.

Fluxo de branches & PR

  1. Crie a branch a partir de develop (ex.: feature-xxxx / fix-...).
  2. Faça commits focados.
  3. Abra um Pull Request para develop. Mantenha os PRs com escopo de uma feature/correção.
  4. Descreva o que mudou e como você validou.

As releases são publicadas a partir de tags de versão na main; o pipeline de publicação só roda para uma tag de release cujo commit esteja na main.

Adicionar um novo domínio (o padrão)

  1. Coletor — crie Collectors/MyThing.ps1 com Get-AerMyThing retornando um [pscustomobject].
  2. Conecte em Public/Invoke-AerReport.ps1 (quatro pontos de toque): o array $collectors, o switch $collectorFile, o switch $data e a chave $reportData.
  3. Página — em Renderer/Dashboard.ps1: adicione uma entrada em $script:AerPages, um navItem na nav lateral e um scaffold HTML em Get-AerPageContent.
  4. Renderização — em Renderer/Assets/Script.ps1: leia d.myThing e construa a UI (reaproveite initDataTable, renderDonut, renderBars, setHeader, kvGrid e os helpers de cell-pill).

Convenções & armadilhas (aprendidas na prática)

  • Alvo PowerShell 7; use apenas chamadas Azure somente-leitura e amigáveis a CancellationToken.
  • Peculiaridade columnar do ARG: o Search-AzGraph pode, de forma intermitente, retornar um único objeto columnar em vez de objetos por linha. Para queries project só com escalares, envolva o resultado com Expand-AerRows (em Core/ResourceGraph.ps1). Não use em queries que legitimamente retornam colunas com valores de array.
  • Recursos com escopo de management group (ex.: policy assignments/exemptions) são descartados por uma query com escopo de assinatura — consulte o endpoint REST do ARG sem filtro de subscriptions (e resultFormat=objectArray) quando precisar do escopo de MG.
  • Palavras reservadas do KQL: não use kind como coluna de saída em project (faça alias, ex.: dcrKind = tostring(kind)).
  • Variáveis automáticas do PowerShell: nunca reutilize $pid ou $host como suas próprias variáveis (são somente-leitura).
  • Arrays JSON: o ConvertTo-Json desembrulha arrays de um único elemento — o lado JS normaliza listas com toArr(); o ConvertFrom-Json do PS7 converte datas ISO automaticamente para [datetime] (formate de forma defensiva).
  • Batch ARM: use Invoke-AerArmBatch (≤20 requisições por chamada) para REST por recurso (ex.: diagnostic settings).

Validar antes de commitar

[System.Management.Automation.Language.Parser]::ParseFile('path\to\file.ps1',[ref]$null,[ref]([ref]$errs).Value)
