Arquitetura & segurança

Como o AER coleta dados, renderiza o relatório e se mantém estritamente somente-leitura e totalmente offline.

Layout do módulo

O AER é um módulo PowerShell carregado por dot-sourcing, com separação clara entre coleta, orquestração, renderização e exportação:

AER.psd1 / AER.psm1          Manifesto do módulo + loader (faz dot-source de tudo)
Public/Invoke-AerReport.ps1  Orquestrador (contexto → coleta paralela → renderização)
Core/ResourceGraph.ps1       Helper de consulta ARG, expansor de linhas columnar-safe, batch ARM, agregadores
Collectors/*.ps1             Uma função Get-Aer* por domínio → retorna um objeto simples
Renderer/
  Dashboard.ps1              Registro de páginas, nav lateral, scaffolds HTML por página, shell do documento
  Export.ps1                 Geração offline de XLSX/PDF e helpers nativos OpenXML/PDF
  Assets/Script.ps1          O app vanilla-JS em um único IIFE (tabelas, gráficos, diagramas)
  Assets/Style.ps1           A folha de estilos

Fluxo de dados

O fluxo é linear e fácil de entender:

  1. Os coletores consultam o Azure (Resource Graph + ARM REST) e retornam objetos PowerShell.
  2. O orquestrador roda os coletores em paralelo e então serializa a saída deles em window.AerData.
  3. O renderizador emite um arquivo HTML por página mais os assets/ compartilhados (style.css, app.js, data.js).
  4. O exportador cria exports/aer-report.xlsx e exports/aer-report.pdf.
  5. No navegador, o JS lê AerData e constrói a UI no cliente — sem nenhum servidor envolvido.
Azure (ARG + ARM)  →  Coletores  →  Orquestrador  →  window.AerData
                                                  ├→ Renderizador → HTML + assets/
                                                  └→ Exportador   → exports/ (xlsx, pdf)

Camadas de processamento

CamadaPropósito
ColetaObter dados de recursos, monitoramento, governança e segurança do Azure usando chamadas somente-leitura.
ProcessamentoNormalizar os achados em um modelo de relatório consistente e calcular agregados/KPIs.
ApresentaçãoGerar saídas legíveis em HTML, XLSX e PDF para públicos técnicos e de gestão.

Segurança & privacidade

  • Somente-leitura por design. O AER só faz chamadas de leitura/listagem/consulta (Resource Graph + ARM GET / POST …/summarize|queryResults|list). Não executa nenhuma operação de escrita.
  • Seus dados ficam com você. A coleta acontece localmente com as suas credenciais; o HTML resultante embute os dados em assets/data.js. Nada é enviado para lugar nenhum.
  • Offline. O relatório gerado não tem dependências externas e funciona sob file://.
Trate a saída como sensível. O relatório contém metadados do ambiente (nomes de recursos, IDs, configuração). Trate a pasta de saída como você trataria qualquer inventário de infraestrutura.
