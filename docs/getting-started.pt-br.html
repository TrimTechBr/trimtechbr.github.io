Primeiros passos com o AER

Instale o AER — Azure Estate Report e gere seu primeiro relatório Azure somente-leitura e totalmente offline em poucos minutos.

Nesta página

1. Requisitos 2. Instalação 3. Autenticar no Azure 4. Gerar o relatório 5. Testar sem Azure 6. Onde fica a saída Próximos passos

1. Requisitos

  • PowerShell 7.0+ (pwsh) — Windows, macOS ou Linux.
  • Módulos Az do PowerShell: Az.Accounts e Az.ResourceGraph.
  • Uma sessão Azure autenticada via Connect-AzAccount.
  • RBAC do Azure no escopo que deseja relatar:
    • Reader — mínimo, para inventário e a maioria das páginas.
    • Monitoring Reader — recomendado, para dados de observabilidade mais ricos.
    • Para ler configurações de App Service (cobertura de App Insights) a identidade precisa de Microsoft.Web/sites/config/list/Action (ex.: Website Contributor); com Reader puro esse sinal é ignorado de forma graciosa.

2. Instalação

Instale a partir do PowerShell Gallery (recomendado):

Install-Module AER -Scope CurrentUser

Ou clone o repositório para rodar a partir do código-fonte (útil para desenvolvimento):

git clone https://github.com/TrimTechBR/azure-estate-report.git
cd azure-estate-report

# Instale as dependências do Azure (uma vez)
Install-Module Az.Accounts, Az.ResourceGraph -Scope CurrentUser

# Importe o módulo
Import-Module ./AER.psd1

3. Autenticar no Azure

Connect-AzAccount

O AER usa o seu contexto e credenciais do Azure já existentes. Ele só faz chamadas de leitura/listagem/consulta — nada é criado, alterado ou removido.

4. Gerar o relatório

# Relatório para todas as assinaturas a que você tem acesso
Invoke-AerReport -OpenReport

O console mostra o progresso enquanto os coletores rodam e o relatório é renderizado:

5. Testar sem Azure

Para validar o layout ou capturar screenshots sem conectar ao Azure, use o dataset de exemplo embutido:

Invoke-AerReport -SampleData -OutputPath .\aer-sample -OpenReport

6. Onde fica a saída

Por padrão, o relatório é escrito em .\aer-report\:

aer-report/
├── index.html                 # abra este
├── assets/                     # style.css, app.js, data.js (dados embutidos)
└── exports/
    ├── aer-report.xlsx
    └── aer-report.pdf
Atenção: o relatório contém metadados do ambiente (nomes de recursos, IDs, configuração). Trate a pasta de saída como você trataria qualquer inventário de infraestrutura.
