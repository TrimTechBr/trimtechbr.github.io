Getting started with AER

Install AER — Azure Estate Report and generate your first read-only, fully-offline Azure report in a few minutes.

On this page

1. Requirements 2. Install 3. Sign in to Azure 4. Generate the report 5. Try it without Azure 6. Where the output goes Next steps

1. Requirements

  • PowerShell 7.0+ (pwsh) — Windows, macOS or Linux.
  • Az PowerShell modules: Az.Accounts and Az.ResourceGraph.
  • An authenticated Azure session via Connect-AzAccount.
  • Azure RBAC on the scope you want to report:
    • Reader — minimum, for inventory and most pages.
    • Monitoring Reader — recommended, for richer observability data.
    • To read App Service settings (App Insights coverage) the identity needs Microsoft.Web/sites/config/list/Action (e.g. Website Contributor); with plain Reader that signal is skipped gracefully.

2. Install

Install from the PowerShell Gallery (recommended):

Install-Module AER -Scope CurrentUser

Or clone the repository to run from source (useful for development):

git clone https://github.com/TrimTechBR/azure-estate-report.git
cd azure-estate-report

# Install the Azure dependencies (once)
Install-Module Az.Accounts, Az.ResourceGraph -Scope CurrentUser

# Import the module
Import-Module ./AER.psd1

3. Sign in to Azure

Connect-AzAccount

AER uses your existing Azure context and credentials. It only issues read/list/query calls — nothing is created, changed or deleted.

4. Generate the report

# Report for every subscription you can access
Invoke-AerReport -OpenReport

The console shows progress as collectors run and the report renders:

  AER · Azure Estate Report
  ──────────────────────────────────────────────
  ▶ Resolving Azure context…
  ✓ Signed in as you@contoso.com  tenant: contoso.onmicrosoft.com
  ✓ 21 subscription(s) in scope
  ▶ Collecting estate data…  21 collectors · parallelism 4
  ✓ Collected all 21 data sets
  ▶ Rendering HTML report…
  ✓ Report generated  C:\...\aer-report\index.html
  ✓ Excel export      C:\...\aer-report\exports\aer-report.xlsx
  ✓ PDF export        C:\...\aer-report\exports\aer-report.pdf
  ✓ Done.

5. Try it without Azure

To validate the layout or capture screenshots without connecting to Azure, use the built-in sample dataset:

Invoke-AerReport -SampleData -OutputPath .\aer-sample -OpenReport

6. Where the output goes

By default the report is written to .\aer-report\:

aer-report/
├── index.html                 # open this
├── assets/                     # style.css, app.js, data.js (embedded data)
└── exports/
    ├── aer-report.xlsx
    └── aer-report.pdf
Heads up: the report contains environment metadata (resource names, IDs, configuration). Treat the output folder as you would any infrastructure inventory.
