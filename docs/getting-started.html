Getting started

This guide explains how to publish the TrimTechBR website using GitHub Pages.

Docs

Repository Publish Structure Customize

1. Create the repository

Create a public GitHub repository named trimtechbr.github.io under the TrimTechBR organization. For an organization site, this repository name becomes the default GitHub Pages website address.

2. Publish the files

Copy all files from this website package to the repository root and push them to the main branch.

git clone https://github.com/TrimTechBR/trimtechbr.github.io.git
cd trimtechbr.github.io
# copy website files here
git add .
git commit -m "Create TrimTechBR GitHub Pages website"
git push origin main

3. Website structure

/
├── index.html
├── about.html
├── contact.html
├── solutions/
│   ├── index.html
│   └── azure-observability-report.html
├── docs/
│   ├── getting-started.html
│   ├── contribution-guide.html
│   └── roadmap.html
└── assets/
    ├── css/style.css
    ├── js/main.js
    └── img/

4. Customize

  • Update solution pages as projects evolve.
  • Replace placeholder repository links with final TrimTechBR repository URLs.
  • Add screenshots under assets/img.
  • Create new pages under solutions/ for future projects.
